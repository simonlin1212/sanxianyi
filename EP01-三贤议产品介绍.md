# EP01 · 三贤议产品介绍

> 形式：对镜头讲解 + 屏幕录屏演示
> 总时长建议：**90-120s**（B 站 / 抖音长版）；如剪 60s 抖音版裁掉架构段
> 录制顺序建议：先录全部画外音，再分段录屏，最后剪辑组合

---

## 录制全文（旁白稿 · 你直接念）

---

### 【0-10s】开场介绍项目

**画面**：你出镜 / 或黑场 + 3 印章浮现
**字幕**：
> 三贤议
> 让 3 位古人，为你的职场困境真实辩论

**旁白**：
> 「我做了个产品叫**三贤议**——让曾国藩、苏轼、王阳明，3 位古人的大智慧，来帮你解你现在职场或生活里的困境。」

---

### 【10-30s】三贤者各自介绍

**画面**：3 张特写卡（印章 + 名字 + 一行核心理念），逐张切换

**旁白**：

> 「**曾国藩**——笨功夫派的代表。
> 一辈子靠日课、慎独，把一件小事做 30 年不间断。他会问你——
> 你下了多少功夫，到几分了？」

> 「**苏轼**——旷达派的代表。
> 三贬黄州、惠州、儋州，一生没崩溃，把每段流放都活出新东西。他不会让你假装'想开了'，但会带你先安顿心。」

> 「**王阳明**——心学派的代表。
> 知行合一、致良知。他给你的不是方案，是一面镜子，照出你藏在两难里不敢承认的事。」

---

### 【30-50s】架构介绍

**画面**：极简架构图（3 印章 → 4 步流程 → 输出）
**字幕**：
> 真 · 多 Agent 辩论架构
> 每位贤者 = 一个独立 Agent
> 各自立场 → 互相质疑 → 辩论 2 轮 → 主持人总结

**旁白**：
> 「不是 1 个 GPT 给标准答案——
> 我用 **agent team 架构**：每位贤者是一个独立的 Agent，
> 他们看到你的困境后，先各自给出自己的看法，
> 然后**互相质疑、辩论 2 轮**，
> 最后由主持人，把 3 人的共识和真实分歧整理给你。
> 不调和，不和稀泥——你自己选。」

---

### 【50-100s】实操演示

**画面**：屏幕录屏

**旁白**：
> 「下面演示一个真实困境——
> 想跳槽到外地的好机会，月薪能涨 50%；
> 但妈妈身体不好需要照顾，老婆又不想离开。
> 三个人，会怎么吵？」

#### 演示输入文本（你直接复制粘贴）
```
想跳槽到一个外地的好机会，月薪能涨 50%。
但我爸妈在老家，我妈身体不好需要人照顾。
老婆又不想离开现在城市的朋友圈。
```

**录屏顺序**：
1. 输入页打字（流式打字效果，加速）
2. 点击「送给三人评议」
3. 3 人首发气泡依次出现（苏 → 曾 → 王）
4. 第二轮反驳：王阳明 @ 苏轼 用历史钩子
   > 「坡仙自己黄州三年，是等心静了再种地，还是种着种着才静的？」
5. 苏轼让步回应（"老夫认这一刀"）
6. 跳到主持人综合 → closing question 大字定格：
   > 「你到底是想去，还是不敢去？把'机会难得'和'家人需要'这两个理由分开说，哪个是真的，哪个是借口？」

**旁白配合**：
- 在 3 人首发时：「3 个人各给一句立场——苏轼说不是 A 还是 B 的选择题，曾国藩说先把信息摸清楚，王阳明直接戳——你不是不知道，你是不敢承认。」
- 第二轮反驳时：「**第二轮，他们会引用对方刚才的话推进**——王阳明用苏轼自己的黄州历史反驳他，苏轼当场让一步。这才是真辩论。」
- closing 出来时：「最后主持人不给标准答案，把王阳明这一问抛回给你——你自己选。」

## 录制 tips

1. **6 分钟 LLM 跑不动？** 录屏时直接从 hero 点 case 1 卡片（跨城跳槽）—— 跳到 synthesis 是预生成的真实数据，跟脚本里的 closing 完全对应
2. **流式辩论部分**：单独录一段输入页 + 点提交（10s）+ live 视图前 30s（够展示 3 个气泡），剪辑接到 synthesis 页
3. **画外音**：先一气呵成把全部旁白录完（用手机/Mac 内置录音），再分段对剪
4. **BGM**：古风+电子混搭，buildup 在 0-30s，鼓点压第二轮反驳，戛然止住在 closing
5. **字幕颜色统一**：奶白主文 / 朱砂红强调 / 暗金关键词

## 题外话回答

### Q1: 一键三连 + 私信 还是 留言？

**推荐：评论区扣关键词 + 你私信发**

- 抖音算法吃**评论数**，纯私信不增推流；纯留言用户被截胡（其他人看见去复制）
- **最佳模板**："想要源码 → 一键三连 + 评论区扣【源码】 → 我私信发你"
- 这样：评论涨数据（推流权重↑）+ 私信精准转化（防截胡）

操作建议：
- 你抖音 IM 后台手动盯关键词回复（前 100 条粉丝量级，手动够用）
- 涨到 1000+ 评论时再考虑用第三方工具（如简言 / 飞鱼）做关键词自动私信

### Q2: 资料库简配版放 GitHub 给粉丝，可行吗？

**完全可行，强烈推荐——这是经典"开源拉粉"打法**

具体策略：

| 完整版（不发） | 简配版（GitHub 公开） |
|---|---|
| 三人语料 yaml 共 180 条（HIGH 80%） | 每人保留 10-15 条最经典 HIGH 语料（共 ~40 条） |
| Router 完整 prompt | 同 |
| 3 agent 完整 prompt | 同（不动） |
| 主持人 4 prompt | 同 |
| 编排脚本 (`orchestrator.py`) | 同 |
| FastAPI server | 同 |
| UI（hero/input/live/synthesis） | 同 |
| 4 个真实跑通 demo case | **保留 1 个**（跨城跳槽）作为示例 |

**简配版优势**：
- 粉丝能跑通完整产品（虽然引文不够丰富）
- "缩减语料"是非常合理的差异——粉丝知道完整版是付费功能
- GitHub stars 涨粉 + 抖音私信涨粉双向加成
- 你前期投入挖掘 180 条语料的核心 IP 被保护

**简配版 GitHub README 写法草稿**：
```
# 三贤议（简配版）

让曾国藩、苏轼、王阳明真辩论你的困境。

> 简配版含 40 条精选语料，跑得通完整流程。
> 完整版含 180 条 HIGH 语料 + 高质量演示数据，
> 关注作者 [@xxx] 私信领取。

## 一键启动

cp .env.example .env  # 填 MiniMax / GLM key
python3 实施/server.py  # 启 backend
open ui/index.html      # 开 UI
```
