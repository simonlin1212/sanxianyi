## 三、关键产品决策（不可逆）

| 决策 | 选择 | 理由 |
|---|---|---|
| 案例卡片点击 | **直接进 synthesis**（结果），不进 debate（过程） | 客户首要看建议好不好，不关心过程 |
| 辩论过程 | 降级为 synthesis 底部「观看完整辩论过程（流式·进阶）」次要入口 | 录视频/进阶用户用 |
| 长期计划 | **严禁** "30/60/90 天观察"等承诺 | 客户不要被 push 长期任务 |
| 对话措辞 | 禁"用户"二字，agent 称对方用"汝/兄/君" | 古雅端庄，不要网络化 |
| 文案语气 | 端庄派，"三家共议·不予定论·只示照见"；不要"审/吵架"等生硬词 | 区别于 ChatGPT 工具感 |
| Live 视图 | 对话气泡 feed（不是单一 stage 文字） | 6 分钟等待变成"3 人正在为你辩论"的真实戏剧 |
| score 字段 | 内部保留，UI 不展示 | 是 Router 算法字段，不是给读者看的 |
| 注册/付费 | **推迟**到阶段 2 | 先验证视觉+内容能爆 |

## 四、技术栈定档

### 前端（ui/）
- 单 HTML + Tailwind CDN v4 + Alpine.js v3 + marked.js + lxgw-wenkai-webfont
- 单 data.js（4 个 demo case 真实数据）
- custom.css（印章 / 进度条 / 对话气泡 / markdown 样式）

### 后端（实施/）
- FastAPI + uvicorn + asyncio loop
- server.py 包装 moderator.py 流式输出
- 9 种 SSE 事件类型：stage / router_done / opinions_done / r1_rebuttals_done / r1_replies_done / r2_rebuttals_done / r2_replies_done / done / error

### 模型
- 默认：**MiniMax-M2.7**（推理模型，16 次调用 ≈ 3-6 分钟，¥0.06-0.10/次）
- 备选：Anthropic Opus 4.7 / GLM-4.7 / DeepSeek（按 .env 切换）

### 启动
```bash
# Backend (5001)
cd 实施/ && python3 server.py

# Frontend (5500)
preview_start("ui-preview")  # 或 npx http-server ui -p 5500
```

## 六、关键经验记录

### 1. Tailwind v4 reset 顽固
- v4 的 `*, ::before, ::after` 重置 border-color 优先级很高
- 解决：用 `:style` inline style 数据驱动，避免 CSS specificity 战争

### 2. macOS sandbox 权限限制
- preview_start 启动 uvicorn 会被沙箱拒绝（uvloop import 错误）
- 解决：直接 Bash 后台跑 `python3 server.py`，preview 不管理 backend 生命周期

### 3. SSE 流式不需要 EventSource
- 用 fetch + ReadableStream + TextDecoder 自己解析 `data:` 行
- 因为 EventSource 不支持 POST，必须自实现

### 4. MiniMax 推理模型 6 分钟问题
- 16 次调用 × 25-90s/次 = 真实场景 3-6 分钟
- 用对话气泡流式 feed 转化体感（"等"变成"看 3 人推进"）
- M1.2 已实现，用户能 follow 上辩论脉络

### 5. UI/prompt 双层文案约束
- 即使 prompt 写"称对方用汝/兄"，LLM 仍可能输出"用户言"
- 必须**显式禁忌**："禁用'用户'二字"
- 同理"禁 30/60/90 天长期计划"必须显式写

### 6. 数据展示分层
- 案例 metadata（score 等内部字段）→ 不给用户看
- 三人输出（中间态）→ 录视频用
- 主持人最终 markdown → 客户看的核心
- 默认主流程跳过中间态，直接给最终结果

## 七、待解 / 风险

1. **6 分钟耗时**：阶段 1 当特色处理（"深度辩论需要时间"），但短视频反响不好需要切 GLM/Opus/DeepSeek 提速
2. **阿里云部署**：当前都跑在本地（preview server + Bash backend），未上公网
3. **SEO/分享卡片**：meta 已设但 og:image 等还没做
4. **服务降级**：单次 6 分钟内 backend 挂了的兜底逻辑（已部分做）
5. **demo case 数据 vs live case 数据**：synthesis 页用 markdown 渲染处理 live，但 demo case 仍用结构化字段——双路径维护成本
