## 三、视觉产品方向

### 3.1 调性：东方水墨 + 现代 SaaS
- 区别于 ChatGPT 类"灰白对话框"工具感
- 区别于 character.ai 类"二次元立绘"玩具感
- 走 **「庄严 + 现代」**——像看一场仪式感强的辩论赛

### 3.2 配色（基于 EP05 Bold Signal 升级）
- **背景**：`#0a0a0a`（深黑）
- **3 人 accent 色**（用于头像/发言侧边条）：
  - 曾国藩：靛青 `#2c3e50`（笃定、稳）
  - 苏轼：茶绿 `#5d7a4a`（温润、活）
  - 王阳明：朱砂 `#a83232`（锋利、破）
- **正文**：奶白 `#f5f1eb`
- **辅助**：浅灰 `#7a7a7a`

### 3.3 三人视觉化方案（推荐 B）

| 方案 | 优点 | 缺点 |
|---|---|---|
| A 手绘水墨半身像 | 最有辨识度 | 需画师/AI 多轮调；伪造历史画像有法律风险 |
| **B 印章 + 书法字** ✅ | 极简、无版权风险、抗印刷、配色可控 | 辨识度依赖文字 |
| C 抽象符号 | 现代感强 | 用户初见看不出谁是谁 |

**B 方案细节**：
- 曾国藩：篆书"曾"+ 老墨色 + 朱砂印
- 苏轼：行书"苏"+ 飘逸笔触 + 落款印
- 王阳明：楷书"王"+ 锐利字号 + 朱砂章

### 3.4 字体
- **标题**：霞鹜文楷 / 思源宋体（古典）
- **正文**：思源黑体 / Inter（清晰）
- **装饰**：JetBrains Mono（点缀英文 nav / 调试感）
- **3 人发言**：可考虑各用不同字体微调

## 四、4 个页面规格

### 页面 1：落地页 / hero
- 顶部：「**职场三人智囊团**」霞鹜文楷大字
- 副标题："让曾国藩、苏轼、王阳明，为你的困境真实辩论 2 轮"
- 3 个印章式头像水平排开（带 hover 微动效）
- 滚动下来：3 个真实案例预览卡片（点开看完整辩论页）
- CTA："说说你的困境"

### 页面 2：输入页
- 居中大输入框（textarea，最少 50 字提示）
- 占位文字："比如：30 岁产品经理，老婆怀孕，创业 offer 月薪翻倍但风险大..."
- 提交按钮："送给三人评议"
- 旁边小字："信息越具体，辩论越精彩"

### 页面 3：辩论现场（核心，最重投资）
- **顶部进度条**：6 阶段（首发 → 第1轮反驳 → 第1轮回应 → 第2轮反驳 → 第2轮回应 → 综合）
- **左侧 3 人头像列**：当前发言者高亮 + 微动效（0.5s 心跳脉冲）
- **右侧主区**：流式打字机（20-30 字/秒）展示当前发言
- **已完成发言**：折叠成卡片可滚动回看
- **必须的细节**：
  - 阶段切换时左侧 accent 条颜色切换
  - 引文（《家书》/《传习录》/《定风波》）特殊渲染：书名号 + 浅色背景框
  - 辩论交锋节点（X→Y）有视觉箭头连接
  - 移动端：3 人头像横排在顶部，发言流在下面

### 页面 4：综合输出页
- 顶部：主持人开场（引用用户原话）
- 中段：3 人发言精炼版（卡片式，3 个 accent 配色）
- 折叠区：「他们吵了 2 轮（点击展开）」（默认折叠）
- 主持人综合：consensus / disagreement 视觉化对比
- 醒目区："今晚做这件事" + "30 天观察什么"
- 底部：王阳明 mirror_question 大字 + 印章红收尾

## 五、技术栈

### 5.1 前端
**单 HTML + Tailwind CDN + Alpine.js（无构建）**

理由：
- 1 个 HTML 文件即开即用，双击就能在浏览器看
- 部署成本 = 0，未来要部署也只是把文件丢上服务器
- 适合 vibe coder（Simon）调试，不用 npm install / webpack

视觉效果库（CDN 引入）：
- AOS.js（滚动入场动画）
- Splitting.js（文字打字机）
- 字体：Google Fonts（思源黑体/思源宋体/JetBrains Mono）+ 自托管霞鹜文楷

### 5.2 数据来源
**M0.5 实测产物**：
- `实施/outputs/run_20260426_113958_m05r2_case_4.md`（35 岁焦虑·完整 2 轮辩论）
- `实施/outputs/run_20260426_113344_m05r2_case_7.md`（被领导骂哭·listen 模式）
- `实施/outputs/run_20260426_112644_m05_case_8.md`（双重困境·完整 2 轮辩论）
- `实施/outputs/run_*_cli.md`（首次跑通的跳槽案例）

把这些 markdown 转成 JSON 注入 HTML 里做演示数据。

### 5.3 文件结构
```
ui/
├── index.html                  # 落地页（含 4 个 section 切换）
├── css/
│   ├── seal-fonts.css         # 自托管霞鹜文楷
│   └── custom.css             # Tailwind 之外的微调
├── js/
│   ├── data.js                # demo 数据（从 M0.5 markdown 转换）
│   ├── debate-stage.js        # 辩论现场逻辑（流式打字+高亮）
│   └── synth.js               # 综合输出页逻辑
└── assets/
    ├── seal-zeng.svg          # 印章曾
    ├── seal-su.svg            # 印章苏
    └── seal-wang.svg          # 印章王
```
