## 资料层级（按价值）

每个人物文件夹按这套规则编号：

- `00-*`：**金标准 skill**（superman / TopPerson / marketplace 三家最强）—— 直接拿来当 system prompt 的基底
- `01-*` ~ `09-*`：community skill / 原典 / 笔记 / 知识图谱
- `README.md`：每个人物的语料清单

## 整体评估

### ✅ 已经够用
- 每人都有 3 套以上**金标准** perspective skill
- 每人都有原典文本（家书 / 标注版 / 传习录知识库）
- 三人都有现代解读资源（marketplace 已蒸馏 + 林语堂图谱 + 度阴山笔记）

### ❌ 真正的缺口（不是语料，是产品）
- **现代职场场景化包装**——把古人智慧映射到 996 / PUA / 裁员 / 35 岁焦虑等具体困境
- **GitHub 上 26 个仓库没人做这一步** —— 这是 Simon 的护城河

## 不必再补语料

进一步搜索 GitHub / Web 已经边际收益极低。下一步应该是：

1. 用现有语料**写三个 agent 的人设 prompt**（融合 marketplace + superman + TopPerson 三家精华）
2. **设计「职场场景化前置层」**——用户输入"我跟老板谈加薪"→ 转换成三人能回答的"决策核心问题"
3. **写主持人综合层**——提炼三派分歧 + 输出三层动作（今日/七日/三十日）
4. **跑 5-10 个真实困境实测** → 看效果再决定是否继续
